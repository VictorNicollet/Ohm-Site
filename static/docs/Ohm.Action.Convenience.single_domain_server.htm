val Ohm.Action.Convenience.single_domain_server

This function creates a server that corresponds to a single domain. This is the case for default server O.server created for you when you start a new Ohm project. Typical use and signature: 

<{ocaml:let server = Action.Convenience.single_domain_server domain

val single_domain_server : 
     ?secure:bool
  -> ?port:int
  -> ?cookies:string
  -> string
  -> unit server}>
  • secure determines whether the server should accept only HTTPS requests. If unspecified, the server accepts only HTTP requests.
  • port determines on what port the server should expect requests. If unspecified, uses the default port 80 for HTTP and 443 for HTTPS.
  • cookies determines what the cookie restriction domain should be (this effectively sets the value of method cookie_domain in Action.server, the default being the domain name.
  • domain is the domain for which the requests should be accepted. Keep in mind that sub-domains are not handled by this server.
